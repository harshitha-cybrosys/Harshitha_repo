Frequently Asked Questions

Who can see the HR Manager Dashboard vs the Employee Dashboard?

Users with the HR Manager or HR Officer access rights will see the full manager dashboard with all employee KPIs, department analytics, and leave overviews. Regular employees will see the self-service dashboard showing only their own personal data, timesheets, contracts, and skill analysis.

What does the "Broad Factor" represent on the employee dashboard?

The broad factor is a numeric value stored per employee in the hr_employee_broad_factor table. It is used to compute a proportional leave allocation indicator, giving employees and managers a benchmark for leave usage relative to entitlement.

How is "experience" calculated on the employee dashboard?

Experience is calculated as the difference between today's date and the employee's first contract start date (date_start from hr.contract). It is displayed as "X years Y months Z days". If no contract exists for the employee, the field shows empty.

What modules are required to install this dashboard?

This module depends on: hr, hr_holidays, hr_timesheet, hr_payroll, hr_attendance, hr_timesheet_attendance, hr_recruitment, ent_hr_resignation, event, ent_hr_reward_warning, and base. All dependent OpenHRMS enterprise modules must be installed first.

Which Odoo version is this module compatible with?

This module is built and tested for Odoo 19 Enterprise. It has been migrated from Odoo 18 with compatibility fixes for removed models (hr.employee.base), deprecated fields (first_contract_date, department_id column), and eliminated external dependencies (pandas).
