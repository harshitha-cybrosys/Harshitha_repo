Key Highlights

Auto-fill Product Details via UPC

Enter or scan any 12/13-digit UPC barcode to instantly fetch and fill the product name, description, price, category, weight, and image from UPCitemdb.

Automatic Category Hierarchy

Automatically creates full parent-child product category hierarchy from the API response if the categories don't already exist in Odoo.

Smart Weight Unit Conversion

Converts product weight from any unit (kg, g, lb, oz) and respects the system's weight unit configuration (kg or lbs) automatically.

Product Image Auto-Fetch

The product image is automatically downloaded from the UPCitemdb API response and set directly as the product's main image in Odoo.

Duplicate Barcode Detection

Checks for existing barcodes before making the API call and raises a clear validation error to prevent duplicate product entries.

Live UPCitemdb API Integration

Integrates directly with the UPCitemdb REST API. Works with any physical barcode scanner or manual entry in the Odoo product form.

Frequently Asked Questions

What does this module do?

This module allows you to create new products in Odoo by simply entering or scanning a UPC barcode in the product form's barcode field. It calls the UPCitemdb API and auto-fills name, description, sales price, weight, category hierarchy, and product image instantly.

Which barcode formats are supported?

The module supports UPC-A (12-digit) and EAN-13 (13-digit) barcodes — the two most common formats found on consumer products worldwide. Any barcode shorter or longer than these lengths is ignored and no API call is made.

What happens if the UPC is not found in the database?

If the UPC is not found or is invalid, the module raises a clear validation error: "Invalid UPC". The trial API on UPCitemdb allows approximately 100 requests per day. If the daily limit is reached, no data will be returned until the next day.

Does it work with a physical barcode scanner?

Yes. You can use any standard USB or Bluetooth barcode scanner. Scan the barcode on a product directly into the barcode field on the Odoo product form — the onchange fires automatically as soon as the value is entered.

What happens if a barcode already exists in Odoo?

Before calling the API, the module checks if the barcode already exists on another product. If it does, a validation error "Barcode already exist" is shown immediately and no API call is made, preventing duplicate product records.
